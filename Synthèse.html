Synthèse : HTML & CSS

Rôles : HTML et CSS

HTML (HyperText Markup Language) structure le contenu : titres, paragraphes, images, liens, listes — c'est le squelette de la page.

HTML = contenu / structure. Sans HTML il n’y a pas de page.

CSS (Cascading Style Sheets) définit l'apparence : couleurs, polices, marges, alignements, mise en page.

CSS = style / design. Sans CSS la page s’affiche mais sans mise en forme.

Langage de balisage vs langage de programmation

Langages de balisage (HTML, CSS) : descriptifs — ils indiquent comment structurer et styliser l'affichage. Ils ne contiennent pas de logique (pas d'if, pas de boucles).

Langages de programmation (Python, JavaScript...) : exécutent des instructions, permettent la logique, les calculs, les conditions et l'automatisation.

Résumé : HTML/CSS décrivent ; Python/JS réalisent des actions.

Qu'est-ce qu'un CMS ?

Un CMS (Content Management System) permet de créer et gérer un site sans coder entièrement. Il offre une interface visuelle, des thèmes et des extensions.

Exemples :

  • WordPress
  • Wix
  • Shopify
  • Drupal, Joomla

Avantages : facile à utiliser, rapide pour créer, pas forcément besoin de coder.

Inconvénients : moins flexible, risque de lenteur si trop d’extensions, dépendance à la plateforme et problématiques de sécurité si mal configuré.

Cycle de vie d'une application web

  1. Idée / Besoin — définir le problème et les utilisateurs.
  2. Conception — maquettes, spécifications, choix techniques.
  3. Développement — front-end (HTML/CSS/JS) & back-end (serveur, DB).
  4. Tests — correction des bugs, compatibilité mobile, ergonomie.
  5. Mise en ligne — hébergement, déploiement, nom de domaine.
  6. Évolution & maintenance — nouvelles fonctions, corrections et mises à jour.

Un site continue d'évoluer après sa publication.
